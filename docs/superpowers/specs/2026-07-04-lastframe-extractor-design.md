# LastFrame Extractor — 설계 문서

날짜: 2026-07-04

## 목적

영상 파일을 올리면 마지막 프레임(또는 끝부분 근처의 원하는 프레임)을 이미지로 추출해
다운로드할 수 있는 웹앱. 추출한 이미지를 영상생성 AI에 입력해 이어지는 영상을 만드는
워크플로우(continuation)에 사용한다.

## 사용 환경

- 사용자 본인 PC에서 혼자 사용. 배포·서버 없음.
- 파일 하나(`lastframe.html`)를 브라우저(Chrome/Edge 권장)에서 열어 사용.
- 영상은 PC 밖으로 전송되지 않음 (완전 클라이언트 사이드).

## 아키텍처

- 단일 HTML 파일: HTML + CSS + JavaScript 전부 포함, 외부 의존성 0.
- 디코딩: 브라우저 네이티브 `<video>` 요소.
- 캡처: `<canvas>`에 현재 프레임을 원본 해상도로 그린 뒤 PNG로 인코딩.
- 프레임 단위 이동: `requestVideoFrameCallback`으로 실제 표시 프레임의
  `mediaTime`을 추적하고, 프레임 간격을 추정해 정밀 스텝 이동.

## 사용 흐름

1. 영상 파일 드래그&드롭 또는 클릭 선택 (반복 사용 가능).
2. 로드되면 자동으로 마지막 프레임으로 이동해 큰 미리보기 표시.
3. 프레임 선택 컨트롤:
   - ◀ / ▶ 프레임 단위 이동 버튼 (키보드 ←/→ 지원)
   - 영상 끝부분 구간을 확대한 미세 탐색 슬라이더
   - 현재 시각/프레임 위치 표시
4. "PNG 다운로드" 버튼 → `원본이름_frame.png`로 저장.

## 에러 처리

- 브라우저가 디코딩 못 하는 형식 → "지원되지 않는 형식" 안내 메시지.
- `duration`으로 직접 seek 시 일부 브라우저에서 검은 프레임이 나오는 문제 →
  duration보다 약간 앞(마지막 실제 프레임 시각)으로 보정해서 seek.
- 영상이 아닌 파일 드롭 → 안내 메시지.

## 검증

- 실제 영상 샘플로 업로드 → 프레임 이동 → PNG 다운로드까지 브라우저에서 직접 확인.

## 범위 제외 (YAGNI)

- 서버 처리, ffmpeg, 다중 파일 일괄 처리, JPG 등 다른 출력 포맷, 배포.
